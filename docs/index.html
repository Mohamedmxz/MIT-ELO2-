Project Overview

This project predicts future rainfall and temperature trends in Sudan by training machine learning models on historical climate records. The goal is to provide actionable insights for farmers, water managers, and communities that depend on climate stability. Models analyze historical patterns to produce guidance on seasonal trends rather than deterministic operational forecasts.

Key Findings

  • Temperature follows a steady seasonal pattern, making predictions relatively reliable.
  • Rainfall is more irregular; predictions vary by region and month and should be used with caution.
  • Forecasts are intended as guidance to understand trends rather than exact operational plans.

Regional Insights

  • Southern and Western regions show more rainfall fluctuation; the North remains mostly dry and stable.
  • Temperatures are generally stable with slight increases in recent years.
  • Most regions show negative correlations between rainfall and temperature; the North shows a slight positive correlation due to very low rainfall.

Methodology

Data Sources

Historical temperature and rainfall data were collected across multiple coordinates in Sudan and merged into a single cleaned dataset.

Preprocessing

  • Handled missing values and normalized ranges where required.
  • Converted and standardized date columns to YYYY-MM-DD and grouped by year/month.
  • Filtered extreme outliers to reduce the impact of erroneous measurements.

Feature Engineering

Features include month, year, geographic coordinates (or grouped region), and core climate variables to help the model learn seasonal and spatial trends.

Machine Learning Models

Random Forest regressors were trained separately for temperature and rainfall. Training pipelines included splitting, training, evaluating, and saving predictions to a CSV file.

Model Evaluation

Evaluation metrics include RMSE and MAE. Temperature predictions show stable error levels, whereas rainfall predictions show higher error due to sparsity and noise.

Pipeline Summary

Data import → cleaning → split → training → evaluation → export predictions (.csv)

Flaws & Limitations

Even though the dataset was structurally clean, the analysis still has important limitations:

  • Data was collected by coordinates, not by administrative cities

    This can reduce interpretability and may not map to real human-relevant areas.

  • Only rainfall and temperature were used

    Weather depends on many additional variables: humidity, pressure, wind, elevation, evaporation, soil moisture, etc.

  • Limited data quantity

    More years or higher-frequency data (daily) would improve model reliability.

  • Potential for additional feature engineering

    Climate indices (ENSO, NAO), satellite data, or spatial features could improve predictions.

  • Large variation in precipitation model performance across regions

    Some regions reached acceptable R2 values near 0.8, while others dropped to 0.4–0.5, showing inconsistent capture of regional rainfall dynamics.

Communication Strategy

  • Use simple language to describe complex climate patterns.
  • Prefer visuals and charts over raw numbers for public audiences.
  • Highlight real-world use-cases targeted at farmers and local water managers.
  • Avoid unnecessary machine-learning jargon — focus on insights.
  • Present seasonal changes clearly and explain uncertainty and limitations.
